# ▶ STAGE 1 — Дизайн-система, перенос React-UI-кита, каркасы
Выполни ТОЛЬКО Stage 1. Foundations §5, §5а — закон.

**Цель:** единые токены и `ui-kit` на Angular+Material+Tailwind, перенесённый с React (без багов, адаптивный), плюс оболочки трёх приложений.

**Сделай:**
- `packages/design-tokens`: `tokens.json` (цвета RGB-каналами + spacing/radius/typography; часть значений вытащи из React-кита) → генерация `tokens.css` (`:root{--color-*}`). Tailwind-конфиг ссылается на `var(--…)`; Angular Material 3 — проброс в `--mat-sys-*`; экспорт TS-объекта для mobile.
- **Перенос React-UI-кита** из `/_input/react-ui-kit/` в `packages/ui-kit`: каждый React-компонент → Angular **standalone**. Маппинг: `useState/useReducer`→signals; props→`input()`; колбэки/события→`output()`; `children`→`ng-content`; условный рендер→`@if/@for`; контекст→Angular DI/сервис. Где есть аналог — Angular Material; стили — Tailwind через токены (никаких хардкод-цветов).
- **Исправь баги исходника** (типизация, состояния) и **доведи респонсивность** (mobile-first, брейкпоинты, без overflow). Добавь a11y (роли, фокус, контраст) и 3 языка.
- Шеллы навигации `main-web`/`vendor-web`/`admin-web`; переключатель языков hy/ru/en; gate входа по роли.
- Сверка с Figma; при расхождении приоритет Figma, расхождения зафиксируй в `ui-kit/MIGRATION_NOTES.md`.

**Тесты:** компонент-тест на каждый перенесённый компонент (рендер, inputs/outputs, состояния); тест «смена токена меняет и Tailwind-, и Material-значение»; тесты адаптивности (брейкпоинты/overflow); тест переключателя языка.
