# ▶ STAGE 15 — Кабинет продавца (vendor-web)
Выполни ТОЛЬКО Stage 15. Foundations §14, §15.

**Сделай:** `feature/vendors`, `vendor-members`, `vendor-bulk`, `analytics`, `subscriptions`.
- Товары: CRUD с обложками фото/видео.
- **Массовые действия** (`vendor-bulk`): выбор множества товаров → изменение цены (на %/на сумму/установка значения), применение/снятие скидки, изменение остатка, активация/скрытие, смена категории; превью затронутых + подтверждение; большие наборы — фоновой задачей (BullMQ).
- Заказы и статусы доставки (`vendor_orders`).
- **Суб-аккаунты и права** (`vendor_members`, `PermissionsGuard`): owner назначает права `orders.view|products.manage|analytics.view|chat.handle|invoices.create`.
- Подписка: план free/pro (без функц. ограничений пока).
- **Аналитика** (право `analytics.view`, scoped по вендору): выручка/GMV по периодам, средний чек, воронка статусов, топ-товары по продажам/просмотрам, конверсия, рейтинг во времени, остатки/алерты, экспорт CSV/Excel; aggregation pipelines + кеш.
- Вендор-UI чата и создания инвойсов (из Stage 8/9) подключи здесь.

**Тесты:** юнит bulk-операций (корректность+превью); юнит агрегатов аналитики; e2e ролей суб-аккаунтов (member без права заблокирован); e2e смены статусов заказа.

**✓ Чек-лист:**
- [ ] продавец ведёт товары/заказы; массовые действия (вкл. фоновые) работают
- [ ] права суб-аккаунтов соблюдаются; аналитика корректна и scoped; экспорт работает
- [ ] тесты/покрытие ок
