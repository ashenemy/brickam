# ▶ STAGE 12 — Лояльность (конструктор + применение)
Выполни ТОЛЬКО Stage 12. Foundations §11 (включая ★-дефолт), §15.

**Сделай:** `feature/loyalty`. Модель `loyalty_programs` (`basis: total_spend|order_count`, `tiers[{level,name,threshold,discountType,discountValue}]`). Вычисление текущего уровня покупателя = макс. tier, чей `threshold ≤ метрика`. Авто-применение скидки на оформлении к сумме после товарных скидок (percent — от суммы, amount — фикс AMD, итог ≥0). **Скидку несёт платформа** (комиссия вендора считается ДО скидки лояльности — не меняй payout вендора). `loyalty_ledger` + обновление `users.loyalty.{totalSpend,totalOrders,currentTierId}` при завершении заказа. UI баланса/уровня в `main-web`. Редактор программы — в Stage 17.

**Тесты:** юнит выбора уровня по метрике; юнит расчёта скидки (percent/amount, неотрицательность, взаимодействие со скидкой товара и комиссией — payout вендора не падает); e2e «заказ→рост метрики→новый уровень→скидка на следующем заказе».
