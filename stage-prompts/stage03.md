# ▶ STAGE 3 — Шаблоны и уведомления
Выполни ТОЛЬКО Stage 3. Foundations §14, §10.

**Цель:** тексты писем/SMS не в коде с самого начала.

**Сделай:** `feature/templates` + `feature/notifications`.
- `templates` (коллекция из §15): email/sms, мультиязычные `content{hy,ru,en}` (+`subject` для email), белый список `variables`, версия. CRUD-бэкенд (UI-редактор будет в Stage 17).
- Шаблонизатор (Handlebars/Liquid): рендер с подстановкой и валидацией переменных (неизвестная/пропущенная → ошибка `ValidationError`).
- `notifications`: интерфейсы каналов (`SmsChannel`, `EmailChannel`) за провайдерами; тело берётся ТОЛЬКО из templates по `key`. OTP-SMS из Stage 2 переключи на шаблон.
- Дефолтные шаблоны (OTP, подтверждение заказа, статус заказа, инвойс, сброс пароля) — добавятся и в сид (Stage 14), но базовые создай здесь.

**Тесты:** юнит рендера+валидации переменных; тест «OTP/уведомления берут текст из templates, не из строк»; рендер на 3 языках.

- [ ] каналы за провайдер-интерфейсами (mock в тестах)
