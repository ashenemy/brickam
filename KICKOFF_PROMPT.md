# BuildHub — Kickoff-промпт для Claude Code

Ты — Claude Code, строишь маркетплейс стройматериалов **BuildHub** (Армения) в Nx-монорепо с нуля.

## Входные документы (прочти ПОЛНОСТЬЮ перед стартом)
1. `buildhub_foundations_v8.md` — постоянный справочник: архитектура, конвенции, бизнес-правила, полная модель данных, настройки. Это закон.
2. `buildhub_stages_v8.md` — план из 20 стейджей (0–19). У каждого: Цель · Задачи · Тесты · Чек-лист (✓).
3. Стартовые конфиги (в `buildhub-starter/`) — положи в корень репо при инициализации:
   `biome.json`, `config/*.toml`, `.env.example`, `commitlint.config.cjs`, `lefthook.yml`,
   `tools/hooks/single-scope-guard.mjs`, `nx.json` (layout+targets), `eslint.boundaries.snippet.mjs`,
   `tsconfig.base.strict.snippet.json`, `TOOLING.md`.

## Доступы, которые нужно запросить у меня, если их нет
- **React UI-kit** (исходник) — ожидается в `/_input/react-ui-kit/`. В Stage 1 переносишь его на Angular+Material+Tailwind (Foundations §5а): standalone-компоненты, signals/input/output, Material где есть аналог, Tailwind через дизайн-токены, **исправить баги и респонсивность**, a11y, 3 языка.
- **Figma-макет** — ссылка/экспорт/скриншоты (или коннектор). При расхождении с React-китом приоритет у Figma.
- **Внешние ключи** (SMS/LLM/embeddings/image/storage/payment) — не выдумывай: провайдер за интерфейсом + `[providers]`+`.env.example`+mock, и скажи мне, что подключить.

## Как работать (обязательно)
1. Идёшь **по одному стейджу за итерацию**, по порядку из `stages_v8`.
2. Соблюдаешь буквально: `type` вместо `interface`; максимум ООП; типы модуля в `src/@types/index.ts`; **никакого хардкода** (всё через TOML/env или админ-настройки в БД); переиспользуемые DTO (одна сущность → mapped-types); generic CRUD/пагинация; глобальные interceptors+`AllExceptionsFilter`+кастомные ошибки; Swagger; Nx tags+границы (feature не импортирует feature).
3. **Форматирование — Biome** (4 пробела, одинарные кавычки), **линт — Biome+ESLint без конфликтов** (см. `TOOLING.md`), Prettier не ставить.
4. **Каждый коммит изолирован одним скоупом** (один app/один package), Conventional Commits с обязательным scope = имя проекта; хук-guard это проверяет.
5. **Тесты — в каждом стейдже** (юнит+критичные e2e), порог покрытия ≥80% в CI; стейдж закрыт только при всех ✓ из чек-листа.
6. После каждого стейджа: зелёная сборка, чистые Biome+ESLint, целые границы Nx, открытый `/api/docs`, отчёт по чек-листу. Затем жди моего «дальше» (или продолжай, если я разрешил автопрогон).

## Старт
Начни со **STAGE 0** (каркас+DevOps-фундамент). Сначала инициализируй Nx-воркспейс (layout `packages/`), внеси стартовые конфиги, затем выполняй задачи Stage 0. По завершении дай отчёт по чек-листу Stage 0 и список того, что нужно от меня для Stage 1 (React-кит, Figma).

## Если хочешь быстрый демо-результат
Спроси меня: идём полным планом или сначала **демо-MVP срез** (Stage 0–5 + 14 сид + база 15). Для первой презентации обычно достаточно среза.
